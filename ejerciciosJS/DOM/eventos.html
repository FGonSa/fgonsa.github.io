EVENTOS

Los eventos son acciones que disparan ciertas funciones de código.

Cuando el usuario presiona un botón, espera un cierto tiempo, redimensiona la pantalla…

Existen una gran variedad de eventos que podemos utilizar para dotar de dinamismo al DOM de nuestras páginas web.

Gracias a estos, podemos conseguir una mejor interactividad con el usuario, y por ende, una mejor experiencia de usuario.

A continuación voy a destacar algunos elementos a tener en cuenta respecto a los eventos:

  • Event Handler

Cuando se activa un evento en JavaScript (como una pulsación de tecla o un movimiento del mouse), se ejecuta un controlador de eventos en respuesta, estos son conocidos como Event Handler. Cada handler se registra en un elemento, conectando el controlador tanto a un elemento como a un tipo de evento (pulsación de tecla, por ejemplo). Un método llamado .addEventListener() actúa como detector de eventos, esperando que ocurra un evento para especificar lo que debe suceder como respuesta y llamar al controlador de eventos.
Por lo general, y resumiendo mucho, los Event Handler son la función a ejecutar cuando se dispara el evento.

  • .addEventListener()

Este método nos permite indicar que esperamos una cierta acción. Cuando dicha acción es recogida, se ejecuta el código programado.
Por ejemplo:

let botonStart = document.getElementById("boton-start");

botonStart.addEventListener('click', ()=>{
alert("Ha sido pulsado el botón Start");
});

El método recoge dos parámetros, el primero, es el evento que tiene que capturar; y el segundo, es la función que ejecutará cuando se produzca el evento.
En este ejemplo, estamos indicando que cuando se pulse (“click”) el botón, se ejecute un alert.
Para dejar de escuchar el evento, se utiliza el método .removeEventListener().

  • Eventos de teclado

Los eventos de teclado describen una interacción del usuario con el teclado. Cada evento describe una interacción separada con una tecla del teclado por parte del usuario, que luego se almacenan con la propiedad .key .

  • keydown los eventos se activan cuando se presiona la tecla por primera vez.

  • keyup los eventos se activan cuando se suelta la tecla.

  • keypress los eventos se activan cuando el usuario presiona una tecla que produce un valor de carácter, esto excluye a teclas como BloqMayús o el Shift.

    Prueba a pulsar la tecla espacio.

  • Eventos de ratón

Un evento de mouse se activa cuando un usuario interactúa con el ratón, ya sea haciendo clic o moviendo el dispositivo.

  • click: los eventos se activan cuando el usuario presiona y suelta un botón del mouse en un elemento.
  • mouseout: los eventos se activan cuando el mouse abandona un elemento.
  • mouseover: los eventos se activan cuando el mouse ingresa o se posiciona sobre el contenido de un elemento. En CSS, sería equivalente a un :hover.
  • mousedown: los eventos se activan cuando el usuario presiona un botón del mouse.
  • mouseup: los eventos se activan cuando el usuario suelta el botón del mouse.
